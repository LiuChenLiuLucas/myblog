title="用户级隔离显卡操作方案"

+++



这是一个非常专业且切中痛点的需求。你不想手动去“抓”进程，而是希望系统能像“自动门”一样，只要检测到是特定用户登录（建立 Session），就自动把他“关”进指定的显卡隔离区。

在 Ubuntu 20.04 上，最成熟、最符合你“监听 Session”想法的方案是利用 **PAM (Pluggable Authentication Modules)** 配合 **`libcgroup-tools`**。

这套机制的原理是：

1. **监听**：PAM 模块会在用户登录（SSH 或 本地登录）时自动触发。
2. **归类**：根据 `/etc/cgrules.conf` 的规则，自动将该用户的所有进程放入指定的 Cgroup。
3. **隔离**：Cgroup 的设备控制器（Device Controller）会根据预设规则，禁止访问特定显卡。

以下是完整的、可落地的操作步骤。

------

### **🛠️ 第一步：安装必要的工具**

我们需要 `libcgroup-tools` 来管理 Cgroup 规则。

```
sudo apt-get update
sudo apt-get install -y libcgroup-tools
```

------

### **📝 第二步：配置 Cgroup 规则（定义“谁能进哪个组”）**

  `/etc/cgrules.conf` 文件。这是 PAM 模块的“地图”，告诉系统哪个用户属于哪个隔离组。

```
sudo vi /etc/cgrules.conf
```

在文件末尾添加以下内容：

```
# 格式：<用户> <控制器> <Cgroup路径>
# 用户A 只能进 grp_012 组
gpu_user012      devices         grp_012

# 用户B 只能进 grp_345 组
gpu_user345       devices         grp_345
```

------

### **🧱 第三步：配置 Cgroup 目录与设备权限（定义“围墙”）**

我们需要创建 Cgroup 目录，并写入具体的设备权限规则（即之前的“先拒绝所有，再允许特定”）。

为了方便管理，我为你写了一个配置脚本，直接运行即可：

```
sudo mkdir -p /sys/fs/cgroup/devices/grp_012
cd /sys/fs/cgroup/devices/grp_012
echo 'a *:* rwm' | tee devices.deny

echo 'c 1:3 rwm' | tee devices.allow
echo 'c 1:5 rwm' | tee devices.allow
echo 'c 5:0 rwm' | tee devices.allow
echo 'c 195:0 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.allow 
echo 'c 195:1 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.allow 
echo 'c 195:2 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.allow 
echo 'c 195:255 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.allow 
echo 'c 249:* rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.allow 

# --- 2. 配置用户 B (显卡 2, 3) ---
# 创建 Cgroup 目录
sudo mkdir -p /sys/fs/cgroup/devices/grp_345
cd /sys/fs/cgroup/devices/grp_345
# 写入规则：先拒绝所有
echo 'c 195:* rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.deny > /dev/null

# 写入规则：只允许 2, 3 和控制设备
echo 'a *:* rwm' | tee devices.deny

echo 'c 1:3 rwm' | tee devices.allow
echo 'c 1:5 rwm' | tee devices.allow
echo 'c 5:0 rwm' | tee devices.allow
echo 'c 195:3 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.allow 
echo 'c 195:4 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.allow 
echo 'c 195:5 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.allow 
echo 'c 195:255 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.allow 
echo 'c 249:* rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.allow

mkdir -p /sys/fs/cgroup/devices/grp_345
mkdir -p /sys/fs/cgroup/devices/grp_01
mkdir -p /sys/fs/cgroup/devices/grp_2345
echo 'c 195:0 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.deny 
echo 'c 195:1 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.deny 
echo 'c 195:2 rmw' | sudo tee /sys/fs/cgroup/devices/grp_345/devices.deny 
echo 'c 195:3 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.deny 
echo 'c 195:4 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.deny 
echo 'c 195:5 rmw' | sudo tee /sys/fs/cgroup/devices/grp_012/devices.deny 

echo 'c 195:0 rmw' | sudo tee /sys/fs/cgroup/devices/grp_2345/devices.deny 
echo 'c 195:1 rmw' | sudo tee /sys/fs/cgroup/devices/grp_2345/devices.deny 
echo 'c 195:2 rmw' | sudo tee /sys/fs/cgroup/devices/grp_01/devices.deny

echo 'c 195:4 rmw' | sudo tee /sys/fs/cgroup/devices/grp_01/devices.deny 
echo 'c 195:5 rmw' | sudo tee /sys/fs/cgroup/devices/grp_01/devices.deny 
```

------

### **⚙️ 第四步：配置 PAM 自动挂载与拦截（核心步骤）**

这是实现“自动监听”的关键。

#### **1. 配置 Cgroup 挂载点**

  `/etc/cgconfig.conf`，告诉系统这些 Cgroup 目录在哪里。

```
sudo vi /etc/cgconfig.conf
```

添加以下内容：

```
# 挂载 devices 控制器
#mount {
#    devices = /sys/fs/cgroup/devices;
#}

# 声明我们要管理的组（必须与 cgrules.conf 对应）
group grp_012 {
    devices {
    }
}

group grp_345 {
    devices {
    }
}
```

#### **2. 启用 PAM 模块**

  `/etc/pam.d/common-session`。

```
sudo vi /etc/pam.d/common-session
```

在文件的**最末尾**添加一行：

```
session optional pam_cgroup.so
```

------

### **🚀 第五步：启动服务**

配置完成后，需要启动守护进程来生效。

1. **启动 Cgroup 配置解析器**（加载 `/etc/cgconfig.conf`）：

   ```
   sudo cgconfigparser -l /etc/cgconfig.conf
   ```

2. **启动规则引擎守护进程**（监听登录事件）：

   ```
   sudo cgrulesengd
   ```

   *注：如果没有报错直接返回，说明启动成功。*

   关闭killall cgrulesengd 2>/dev/null
   sudo cgclear

   ps aux | grep cgrulesengd

   lscgroup

------

### **🧪 第六步：验证（无需手动操作）**

现在，你完全不需要手动输入 PID 了。

1. **重新登录**：
   退出当前的 `grp_012` 会话，然后重新登录（`su - grp_012` 或 SSH 登录）。
   *系统会在登录的一瞬间，自动检测到是 `grp_012`，并根据 PAM 配置把他扔进 `grp_012` 这个 Cgroup。*

2. **验证 Cgroup 归属**：
   登录后直接运行：

   ```
   # 查看当前 Shell 是否在正确的 Cgroup 里
   cat /proc/self/cgroup | grep devices
   ```

   **预期输出**：应该包含 `devices:/grp_012`。

3. **验证显卡隔离**：

   ```
   # 尝试访问禁区显卡 (假设 2 号卡不属于他)
   dd if=/dev/nvidia2 of=/dev/null count=1
   ```

   **预期结果**：
   `dd: failed to open '/dev/nvidia2': Operation not permitted`

### **⚠️ 重要提示：关于重启**

上述配置在重启后会失效（因为 `/sys/fs/cgroup` 是内存文件系统）。为了让这套方案永久生效，你需要把启动命令加入开机自启。

最简单的方法是  `/etc/rc.local`（如果没有就创建一个）：

```
sudo vi /etc/rc.local
```

内容如下：

```
#!/bin/ 
# 等待 systemd 初始化完成
sleep 5
# 重新应用 Cgroup 配置
cgconfigparser -l /etc/cgconfig.conf
# 启动监听守护进程
cgrulesengd
```

然后赋予执行权限：`sudo chmod +x /etc/rc.local`。
